Render Deployment Guide

1. Create a Render Account

Sign up at render.com if you don't have an account

2. Prepare Your Project

Create these files in a new directory:

// Directory structure: ├── server.js ├── public/ │ └── index.html (this file) ├── package.json └── .gitignore

3. Create package.json

Add this content to package.json:

{ "name": "instagram-bot", "version": "1.0.0", "description": "Instagram Automation Tool", "main": "server.js", "scripts": { "start": "node server.js" }, "dependencies": { "express": "^4.18.2", "axios": "^1.7.2", "dotenv": "^16.4.5", "uuid": "^9.0.1" }, "engines": { "node": ">=18.0.0" } }

4. Create server.js

Add this backend code:

const express = require('express'); const app = express(); const PORT = process.env.PORT || 3000; // Serve static files app.use(express.static('public')); // Instagram OAuth endpoint app.get('/auth/instagram', (req, res) => { // OAuth implementation would go here res.redirect('/?session=connected'); }); // Webhook endpoint app.post('/webhook', (req, res) => { // Webhook processing would go here res.sendStatus(200); }); // Start server app.listen(PORT, () => { console.log(`Server running on port ${PORT}`); });

5. Push to GitHub

Create a repository and push your code

6. Deploy to Render

- Create a new Web Service on Render
- Connect your GitHub repository
- Set environment variables in Render dashboard:
  FACEBOOK_APP_ID, FACEBOOK_APP_SECRET,
  REDIRECT_URI, WEBHOOK_VERIFY_TOKEN
- Deploy!
